# 3D 棋盘渲染差异分析（现状 vs 设计图）

## 结论（先回答“能不能做”）

可以做，而且当前代码基础已经具备 70% 以上能力：
- 已使用 `three_js`，并且场景里已经有环境光、方向光、聚光灯、多材质、棋子阴影开关。 
- 现阶段问题不在“有没有 3D 引擎能力”，而在“光照参数、阴影质量、材质贴图/法线、后处理缺失”。

也就是说：**可行，属于参数与材质升级，不是架构重写**。

## 当前实现盘点（代码级）

## 1) 光照（Light）

当前场景已有三层光源：
- `AmbientLight(0xffe2c4, 0.24)`：暖色环境光。
- 主光 `DirectionalLight(0xffd4a2, 0.94)`：已启用 `castShadow=true`。
- 辅助光 `DirectionalLight(0xe9d8c1, 0.15)` + `SpotLight(0xffc784, 0.42, ...)` 用于提亮与局部 sheen。 

这说明你的“方向光”思路和当前代码一致，已经在用。差异是主光方向与强度还没调到设计稿的“午后斜射+柔和高光”状态。

## 2) 阴影（Shadow）

当前已开启部分阴影开关：
- 主方向光 `castShadow = true`。
- 棋子 `castShadow=true` 且 `receiveShadow=true`。
- 棋盘网格/木纹细节（很多是 `MeshBasicMaterial`）主要依赖 baked/伪阴影，不受真实光照。 

关键缺口：
- 未看到对 shadow map 质量参数的显式控制（如 mapSize、bias、radius、camera frustum 等）。
- 棋子“接触阴影”更多由手工椭圆阴影 mesh 模拟，真实投影存在但观感主导仍是 fake shadow。

## 3) 材质（Material）

已有：
- 棋盘顶面/侧面用 `MeshStandardMaterial`（roughness/metalness 已配置）。
- 棋子也是 `MeshStandardMaterial`。 

缺口：
- 木板没有真实纹理贴图（albedo/normal/roughness map），现在是纯色 + 多条 `BoxGeometry` 伪木纹。
- 网格线用 `MeshBasicMaterial`，不参与 PBR 光照。
- 黑白棋 roughness 与设计目标不一致：当前白棋 roughness(0.32) 比黑棋(0.40)更“亮”，设计图通常是黑棋更镜面、白棋更柔和。

## 设计图对比后的实现差异

按你要求聚焦三块：

## A. 光（Lighting）差异

设计图特征：
- 统一、清晰的主光方向（右上/左上斜射之一）。
- 顶面木纹有明显“方向性渐变”，不是平均照亮。
- 棋子高光点更集中。

现状差异：
- 多灯并用但主次关系不够清晰，导致“灰雾化提亮”。
- 聚光灯 + 环境光叠加偏高，削弱了体积感。

建议参数方向：
- 降低环境光（例如 0.24 → 0.12~0.18）。
- 主 DirectionalLight 强度更集中并固定光向；fill 仅补暗部，不抢主光。
- 统一色温：主光偏暖白，fill 偏中性，减少橙色叠光。

## B. 阴影（Shadow）差异

设计图特征：
- 棋子下方接触阴影短而实，远侧有软阴影拖尾。
- 阴影边缘软但不糊，且方向统一。

现状差异：
- 主要靠手工 shadow mesh（固定偏移），在不同相机/光向下不完全可信。
- 真实阴影精细度可能不足（未显式配置 shadow map 细节）。

建议实现：
- 保留接触阴影 mesh（性能友好），但降低其存在感；真实阴影作为主感知来源。
- 为主方向光配置 shadow map：
  - 提升 map 分辨率（如 1024/2048）；
  - 调整 bias 避免 acne；
  - 收紧阴影相机包围盒，集中分辨率到棋盘区域。

## C. 材质（Material）差异

设计图特征：
- 棋盘是浅木材，纤维纹理连续，网格像轻微压印而不是“浮在表面的线”。
- 黑棋有清晰高光，白棋有体积但不过曝。

现状差异：
- 木纹是程序化线段，缺少真实法线扰动与微表面变化。
- 棋子表面参数仍偏“塑料感”，尤其白棋偏亮、黑棋高光不够尖。

建议落地：
- 棋盘顶面引入 PBR 纹理集：
  - `map`（albedo）、`normalMap`、`roughnessMap`（可选 AO）；
  - 网格线可考虑独立 decal/overlay，或在纹理中预烘焙轻压痕。
- 棋子参数建议：
  - 黑棋：`roughness 0.18~0.26`，`metalness 0.02~0.08`；
  - 白棋：`roughness 0.38~0.52`，`metalness 0.0~0.03`。

## 对你“初步分析”的逐条回应

1) **方向光与阴影**：正确，且代码已经有基础。下一步主要是“阴影质量参数 + 主光统一化”。
2) **PBR 材质与高光**：正确。当前虽用了 `MeshStandardMaterial`，但还没进入“完整 PBR 贴图流程”。
3) **焦点与景深**：方向正确，但 Flutter `three_js` 里做后处理链要评估插件能力/性能。建议分阶段：先完成光照与材质，再看 DOF。

## 可执行改造路线（低风险）

### Phase 1（1~2 天）：参数调优，不加贴图
- 收敛光源：降 ambient、稳主光方向、减 spot 权重。
- 明确 shadow 参数（map/bias/frustum）。
- 调整黑白棋 roughness/metalness，先获得正确高光关系。

### Phase 2（2~4 天）：木板 PBR 化
- 接入木板贴图资源（最少 albedo + normal + roughness）。
- 顶面材质改成贴图驱动；侧面可先保留程序材质。
- 网格线从“悬浮线”改成“贴图压印感”或薄 decal。

### Phase 3（可选，1~3 天）：镜头语言与后处理
- 若性能允许，引入轻量 DOF/色调映射调优。
- 增加构图预设（首页静态镜头优先，不做重动画）。

## 风险与注意点

- 纹理资源体积会影响首屏；需控制分辨率并考虑压缩。
- iOS Safari/WebGL 与低端 Android 对阴影和后处理差异较大，需真机比对。
- 当前代码大量使用 `MeshBasicMaterial` 叠加细节，迁移到完整 PBR 后需避免“重复加亮”。

---

## 最小可交付验收标准（建议）

- 主光方向稳定，棋子阴影方向与 UI 视觉一致。
- 黑棋高光明显强于白棋，白棋高光更宽更柔。
- 棋盘顶面可见连续木纹细节（非随机线段感）。
- 与设计图同构图下，观感差异集中在“艺术细节”，不再是“材质级别差一代”。
