# 下棋页 3D 棋盘渲染实现说明（代码级）

> 目标：解释截图中棋盘区域“是怎么被代码构建出来的”，以及“这些实现如何导致当前视觉结果（光泽偏弱、阴影偏弱、反光偏弱）”。

## 1. 页面里的 3D 棋盘是如何挂载的

在下棋首页（`capture_game_screen`）里，3D 棋盘是作为 `Stack` 的背景层放置的：

- 用 `Positioned` 固定在页面上部区域；
- `IgnorePointer` 让它不吃触摸事件；
- 通过参数传入镜头、叶影、粒子等控制项。

也就是说：**它是“视觉背景层”，不是交互棋盘本体**（交互 UI 在上层卡片中）。

## 2. 3D 场景初始化逻辑（three_js）

3D 背景由 `GoThreeBoardBackground` 组件实现，底层使用 `three_js`。
初始化时设置了：

- 透明背景（`alpha: true`）
- 抗锯齿（`antialias: true`）
- `toneMappingExposure: 0.70`（整体曝光偏保守）

`_setup()` 时按顺序创建：

1. Scene
2. Root Group（棋盘整体）
3. Perspective Camera
4. 灯光
5. 棋盘主体
6. 木纹/网格细节
7. 叶影、粒子
8. 棋子
9. 每帧动画更新（镜头漂移、叶影摆动、粒子浮动）

## 3. 镜头（Camera）设定与关系

镜头不是固定死机位，而是带轻微漂移：

- `drift/lift` 用正弦函数让镜头缓慢呼吸；
- `sceneScale` 通过“相对 target 缩放距离”控制推拉；
- cinematic 模式使用更窄 FOV（24）和更贴近的机位参数。

因此棋盘会有轻微电影感透视，不是完全正俯视。

## 4. 光照系统：亮度、方向、关系

场景内核心有三类灯：

### 4.1 Ambient（环境底光）

- 颜色：暖色 `0xffead2`
- 强度：`0.16`
- 用途：抬底，不产生方向性阴影。

### 4.2 Key Directional（主光）

- 颜色：`0xffd7a4`（暖）
- 强度：`1.02`（动画时会脉冲到约 `0.98 * intensityPulse`）
- 位置：`(4.8, 6.0, 4.2)`（右前上方）
- 目标点：`(0.8, -0.10, -0.55)`
- `castShadow = true`（允许阴影）

### 4.3 Fill + Sheen（辅光 + 光泽补光）

- Fill Directional：强度仅 `0.10`，主要补暗部；
- Sheen Spot：强度 `0.16`，用于局部“暖高光氛围”。

**灯光关系总结**：

- 主光决定主要明暗方向；
- 环境光和辅光把暗部抬起来，降低反差；
- Spot 只做轻微提亮。

结果是整体柔和、偏暖，但高光冲击和阴影反差不强。

## 5. 棋盘模型是怎么来的（几何拼装）

棋盘不是外部模型导入，而是代码拼出来：

- 两个长方体 + 四角圆柱，组成厚棋盘主体与圆角；
- 顶部再叠一层薄“皮”，形成上下材质层次；
- 前缘再加一条半透明 `frontGlow` 提氛围。

### 材质参数（关键）

- 侧面：`MeshStandardMaterial`，`roughness: 0.78`
- 顶面：`MeshStandardMaterial`，`roughness: 0.56`
- `metalness` 都接近 0

**影响**：粗糙度偏高会让高光更散、更暗，棋盘反光感偏弱。

## 6. 网格、木纹、边缘细节怎么做的

棋盘表面的线和木纹主要不是贴图，而是“很多细几何 + 透明材质”：

- 网格线：细 Box；
- 网格暗槽/亮槽：额外两层更细的 Box 叠加；
- 木纹：大量随机长度细条；
- 侧面纹理：前侧细条叠加。

这些细节大量使用 `MeshBasicMaterial`（不受光照影响），属于视觉装饰层，不是物理反射层。

## 7. 阴影是怎么来的

当前阴影来源有两类：

### 7.1 真实灯光阴影（有限）

- 主方向光开了 `castShadow`；
- 棋子 mesh 开了 `castShadow/receiveShadow`；
- 棋盘几何开了 `receiveShadow`。

### 7.2 人工“假阴影”

每个棋子下方额外放一个扁平深色 Cylinder（透明、略偏移、椭圆缩放），用于强化接触阴影观感。

另外还有“叶影”系统：在棋盘上方铺一组半透明圆片（core + penumbra）并做缓慢摆动，形成右上区域那种斑驳暗影。

## 8. 棋子（截图里提到“妻子”，这里按“棋子”）怎么来的

每颗棋子由两部分构成：

1. 阴影片（扁平 Cylinder）
2. 棋子本体（Sphere 后在 Y 轴压扁）

关键参数：

- 半径 `radius = step * 0.46`
- 形体：`SphereGeometry(radius, 48, 18)` + `scale.y = 0.34`（围棋子扁圆造型）
- 黑子材质：`roughness 0.20 / metalness 0.06`
- 白子材质：`roughness 0.46 / metalness 0.02`

**视觉结果**：

- 黑子有一点高光，但偏柔；
- 白子粗糙度更高，“瓷感亮斑”更弱；
- 阴影主要依赖假阴影 + 轻真实阴影，接触感不算重。

## 9. 环境/反光为什么偏弱（结构性原因）

从代码结构看，这套场景是“程序化暖色风格”，并非 IBL 写实链路：

- 没看到 HDRI / envMap / CubeCamera 反射流程；
- 材质 roughness 整体偏高；
- 曝光（0.70）和补光策略偏柔和；
- 大量装饰层是 `MeshBasicMaterial`，不参与真实光照计算。

所以会直接出现：

1. 棋子光泽不够
2. 棋子阴影不够
3. 棋盘反光不够

## 10. 对照截图可见现象

- 右上“斑驳暗影”= 叶影圆片系统（非真实树叶投影）
- 棋子下“软糊阴影”= 人工扁平阴影片为主
- 棋盘高光不强 = 顶面粗糙度偏高 + 无明显环境反射管线
- 整体暖而平 = 环境光 + 弱补光组合抬底

## 11. 一句话结论（给技术美术）

当前实现是：**程序化几何 + 暖色柔光 + 人工氛围层（假阴影/叶影）**。

它能快速产出风格化背景，但在“镜面反射、硬接触阴影、强高光层次”上天花板较低，这正是当前观感问题的主要来源。
